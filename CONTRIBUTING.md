# Contributing to Digital Ally

Thank you for contributing to the ELUSOC open-source event. This guide helps
first-time contributors and returning maintainers work smoothly.

## Code of Conduct

Please be respectful and inclusive. We follow the GitHub Community Guidelines.

## Quick Start

1. Fork the repository and clone your fork.
2. Create a branch: `git checkout -b feature/short-description`.
3. Install dependencies: `npm install`.
4. Run the dev server: `npm run dev`.
5. Commit your changes with a clear message.
6. Open a pull request.

## Contribution Types

- Bug fixes
- New features or improvements
- Documentation updates
- Tests and tooling
- UI/UX enhancements

## Issue Assignment

If you assign yourself to an issue, please post a brief update within 20 hours.
After 24 hours of inactivity, the assignee may be removed so others can help.

## ELUSOC Onboarding

- Introduce yourself in the issue before starting work.
- Share progress updates to help maintainers coordinate.
- If you are new, look for the `good first issue` label.

## GSSoC and Hacktoberfest

We welcome seasonal program contributors. Please follow the contribution flow
and keep PRs focused and well documented. Maintain a clear link to the issue
or request you are addressing.

## Pull Request Checklist

- Keep changes scoped to a single topic.
- Update documentation when behavior changes.
- Add or update tests when needed.
- Ensure `npm run build` succeeds.

## Getting Help

Open a question issue or start a discussion if you get stuck.
